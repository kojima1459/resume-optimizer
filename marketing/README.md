## 1兆トークンキャンペーン応募文

### 投稿文（渾身の1パターン）

```
転職活動で一番辛いのは「書類選考で落ちる理由がわからない」こと。

そこで作ったのが、求人票に合わせて職務経歴書をAIが最適化するツール。

Manusの力で実現した3つの革新：

1. 複数パターン生成→AI評価→ベスト選択
   一度に3パターン生成し、求人との適合度をスコアリング。もう「どれがいいか」で悩まない。

2. ドラッグ&ドロップで秒
   PDF/Word/テキストをドロップするだけ。OCRでテキスト抽出、入力の手間ゼロ。

3. バッチ一括応募機能（開発中）
   1つの職務経歴書を複数の求人票に対して一度に最適化。転職活動の効率が10倍に。

技術スタック：
- tRPC + React 19で型安全な開発
- Gemini API + Built-in LLMのハイブリッド
- S3統合でファイル管理
- Manus OAuthで認証

転職活動の「書類選考で落ちる」という痛みを、Manusの完全な機能を活用して解決しました。

#BuiltwithManus

🔗 [アプリURL]
```

### 評価基準への対応

#### 創造性（アイデアの独自性とユニークさ）
- 既存の職務経歴書作成ツールは「一般的な職務経歴書」を作成するのみ
- 本ツールは「求人票に合わせて最適化」という独自のアプローチ
- 複数パターン生成＋AI評価という新しい体験

#### 実用性（現実世界での実用性と影響）
- 転職活動中の求職者に直接的な価値を提供
- 「書類選考で落ちる」という具体的な痛みを解決
- 時間の節約と応募成功率の向上

#### 実行（ビルドの品質、デザイン、ユーザーエクスペリエンス）
- tRPC + React 19で型安全な開発
- 直感的なUI/UX、レスポンシブデザイン
- 多言語対応（日本語・英語）

#### Manusの熟練（Manusの完全な機能を効果的に活用）
- LLM統合（Gemini API + Built-in LLM）
- ファイルストレージ（S3統合）
- データベース統合（MySQL/TiDB）
- 認証機能（Manus OAuth）
- tRPC統合（型安全なAPI）
