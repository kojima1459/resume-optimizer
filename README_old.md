# 職務経歴書最適化ツール

**求人情報に合わせて、あなたの職務経歴書を最適化します**

職務経歴書最適化ツールは、AI（大規模言語モデル）を活用して、応募する求人情報に合わせた職務経歴書の各項目（職務要約、志望動機、自己PR、なぜ御社かなど）を自動生成するWebアプリケーションです。ユーザーは自分の職務経歴書と求人情報を入力するだけで、応募先企業に最適化された文書を簡単に作成できます。

## 主な機能

### 基本機能

職務経歴書最適化ツールは、以下の基本機能を提供します。

**職務経歴書と求人情報の入力**では、ユーザーはテキスト入力フォームまたはファイルアップロード（PDF、Word）を通じて、自分の職務経歴書と応募する求人情報を入力できます。画像ファイルからの求人情報抽出にも対応しており、OCR（光学文字認識）技術を使用してテキストを自動抽出します。

**出力項目の選択**機能では、職務要約、志望動機、自己PR、なぜ御社か、職務経歴（最適化版）、企業で実現したいことなどの標準項目から、生成したい項目を選択できます。さらに、カスタム項目を動的に追加することも可能で、ユーザー独自のニーズに柔軟に対応します。

**文字数指定**機能により、各項目の文字数を指定して生成できるため、応募先企業のフォーマットに合わせた文書作成が可能です。

**AI生成**では、Manus内部のLLMまたはユーザーが設定したGemini APIキーを使用して、求人情報に最適化された各項目を自動生成します。

### 高度な機能

**個別項目の再生成**機能により、生成後に特定の項目だけを再生成できます。これにより、満足のいく内容になるまで調整が可能です。

**項目別編集機能**では、生成後にテキストエリアで各項目を自由に編集できます。AIが生成した内容をベースに、ユーザー自身が微調整を加えることができます。

**コピー機能**として、個別項目のコピーと全項目の一括コピーの両方に対応しています。生成した内容を簡単に他のアプリケーションに貼り付けることができます。

**英語翻訳機能**では、生成した各項目を英語に翻訳できます。外資系企業への応募や英文レジュメの作成に便利です。

**一括ダウンロード機能**により、生成した全項目をWord形式またはPDF形式でダウンロードできます。

**複数パターン生成機能**では、1回の操作で複数パターン（最大10パターン）の文書を生成し、その中から最適なものを選択できます。

### データ管理機能

**生成履歴の保存**機能により、過去に生成した職務経歴書の履歴をデータベースに保存します。ユーザーは後から履歴を参照し、再利用することができます。

**履歴一覧表示・詳細表示**では、保存された履歴を一覧表示し、詳細を確認できます。

**履歴の再利用**機能により、過去の履歴を読み込んで再編集や再生成が可能です。

**履歴の削除**機能で、不要な履歴を削除できます。

### カスタマイズ機能

**APIキー設定機能**では、ユーザーが独自のGemini APIキーを設定できます。設定されたAPIキーは暗号化してデータベースに保存され、セキュリティが確保されます。

**業界別・職種別テンプレート機能**により、IT、金融、製造、営業、マーケティングなどの業界・職種別に最適化されたプロンプトテンプレートを選択できます。これにより、より精度の高い文書生成が可能になります。

**ユーザー独自テンプレートのカスタマイズ機能**では、ユーザーが独自のプロンプトテンプレートを作成・保存・管理できます。繰り返し使用するシナリオに対応し、効率的な文書作成をサポートします。

## 技術スタック

### フロントエンド

フロントエンドは、React 19を使用して構築されています。UIコンポーネントライブラリとしてshadcn/uiを採用し、スタイリングにはTailwind CSS 4を使用しています。ルーティングにはwouterを使用し、軽量で高速なページ遷移を実現しています。

### バックエンド

バックエンドは、Express 4をベースとしたNode.jsサーバーで構築されています。tRPC 11を使用してエンドツーエンドの型安全なAPIを実現し、フロントエンドとバックエンド間の通信を効率化しています。

### データベース

データベースには、MySQL/TiDBを使用しています。ORMとしてDrizzle ORMを採用し、型安全なデータベース操作を実現しています。

### 認証

認証には、Manus OAuthを使用しています。ユーザーはManusアカウントでログインし、安全に職務経歴書最適化ツールを利用できます。

### AI統合

AI統合では、Manus内部のLLMまたはGemini APIを使用しています。ユーザーは設定画面で独自のGemini APIキーを設定することも可能です。

### その他

OCR（光学文字認識）には、Tesseract.jsを使用しています。ファイル処理では、PDFやWord文書からのテキスト抽出に対応しています。

## セットアップ手順

### 前提条件

セットアップには、以下のソフトウェアが必要です。

- Node.js 22.13.0以上
- pnpm（パッケージマネージャー）
- MySQL/TiDBデータベース

### インストール

リポジトリをクローンします。

```bash
git clone https://github.com/yourusername/resume_optimizer.git
cd resume_optimizer
```

依存関係をインストールします。

```bash
pnpm install
```

### 環境変数の設定

プロジェクトルートに`.env`ファイルを作成し、以下の環境変数を設定します。

```env
DATABASE_URL=mysql://user:password@host:port/database
JWT_SECRET=your-jwt-secret
VITE_APP_ID=your-manus-app-id
OAUTH_SERVER_URL=https://api.manus.im
VITE_OAUTH_PORTAL_URL=https://auth.manus.im
OWNER_OPEN_ID=your-owner-open-id
OWNER_NAME=your-owner-name
VITE_APP_TITLE=職務経歴書最適化ツール
VITE_APP_LOGO=/logo.svg
BUILT_IN_FORGE_API_URL=https://forge-api.manus.im
BUILT_IN_FORGE_API_KEY=your-forge-api-key
VITE_FRONTEND_FORGE_API_KEY=your-frontend-forge-api-key
VITE_FRONTEND_FORGE_API_URL=https://forge-api.manus.im
```

### データベースのマイグレーション

データベーススキーマを作成します。

```bash
pnpm db:push
```

テンプレートデータをシードします。

```bash
node seed-templates.mjs
```

### 開発サーバーの起動

開発サーバーを起動します。

```bash
pnpm dev
```

ブラウザで`http://localhost:3000`にアクセスすると、アプリケーションが表示されます。

## 使用方法

### 基本的な使い方

職務経歴書最適化ツールの基本的な使い方は以下の通りです。

まず、Manusアカウントでログインします。ログインボタンをクリックし、Manus OAuthでログインしてください。

次に、職務経歴書を入力します。テキスト入力フォームに直接入力するか、PDF/Wordファイルをアップロードしてください。

求人情報を入力します。テキスト入力フォームに直接入力するか、PDF/Wordファイル、または画像ファイルをアップロードしてください。

テンプレートを選択します（オプション）。業界別・職種別テンプレートまたはユーザー独自テンプレートから選択できます。

出力項目を選択します。職務要約、志望動機、自己PR、なぜ御社かなどの項目から、生成したい項目を選択してください。カスタム項目を追加することもできます。

文字数を設定します。各項目の文字数を指定してください。

「生成開始」ボタンをクリックすると、AIが最適化された文書を生成します。

生成結果を確認し、必要に応じて編集や再生成を行います。

最後に、コピーまたはダウンロードして、応募書類として使用してください。

### テンプレートのカスタマイズ

ユーザー独自のテンプレートを作成するには、以下の手順を実行します。

「マイテンプレート」ページにアクセスします。

「新規作成」ボタンをクリックします。

テンプレート名、説明、プロンプトテンプレートを入力します。プロンプトテンプレートには、`{{resumeText}}`と`{{jobDescription}}`を使用すると、入力内容が自動的に埋め込まれます。

「作成」ボタンをクリックすると、テンプレートが保存されます。

作成したテンプレートは、ホーム画面の「テンプレートを選択」セクションで選択できます。

### APIキーの設定

独自のGemini APIキーを設定するには、以下の手順を実行します。

「設定」ページにアクセスします。

「Gemini APIキー」フィールドにAPIキーを入力します。

「保存」ボタンをクリックすると、APIキーが暗号化されてデータベースに保存されます。

設定後、生成時に独自のGemini APIキーが使用されます。
